
Este software foi desenvolvido com a intensão de sanar o problema que temos com o web service dos Correios. Funcionando como um espelho, tendo o serviço disponível sempre que a sua loja também estiver.
Também pode ser bom para economizar consultas no web service da(s) transportadora(s). Consultas que ocorrem a todo momento, na pré-compra, que seriam via Internet, passam a ser consultas no localhost ou "offline".

Outra possibilidade é usa-lo também com web services de transportadoras, basta configurar a atualização do método no painel administrativo, informando a URL do WS, e uma Skin XML compatível com o WS da transportadora.

A interface administrativa possibilita a criação de CEPs de Origem, Métodos de Serviço com algumas configurações, e monitora as atualizações que devem ser executadas.

A base de dados padrão acompanha somente os métodos SEDEX e PAC. Para inserir SEDEX 10 por exemplo, basta cadastrar um novo método com o código correspondente(verificar o do "sedex 10") e em seguida atualizar os dados para este método (que será sugerida no painel de atualizações)
